## DOMAIN KNOWLEDGE

1. **Инструменты трекинга**:
   - **GA4**: event-based модель (не session-based как в UA). Ключевые концепции: Events, Parameters, Conversions, Audiences, Explorations. BigQuery export для сырых данных. Engagement Rate вместо Bounce Rate.
   - **GTM**: контейнер → теги (что отправляем) → триггеры (когда срабатывает) → переменные (какие данные передаём). Data Layer — центральная шина для передачи данных между сайтом и тегами. Preview Mode для отладки.
   - **Segment**: CDP (Customer Data Platform — единая точка сбора данных). Принцип: один раз описал события → распределяет в GA4, Amplitude, CRM и т.д. Sources → Destinations.
   - **Mixpanel**: сильный в продуктовой аналитике. Funnels, Retention, Flows. Хранит event-данные, не агрегаты.
   - **Amplitude**: аналог Mixpanel. Лучше для больших объёмов. Chart types: Event Segmentation, Funnel Analysis, Retention, Pathfinder.

2. **UTM параметры**:
   - Обязательные для всех платных каналов: `utm_source` (площадка: google, facebook, vk), `utm_medium` (тип: cpc, email, social), `utm_campaign` (название кампании), `utm_content` (вариант объявления/баннера), `utm_term` (ключевое слово для поиска).
   - Naming convention: snake_case, нижний регистр, без пробелов. Формат кампании: `[product]_[geo]_[goal]_[date]`. Пример: `utm_campaign=saas_ru_trial_2026q1`.
   - Ошибки: прописные буквы, пробелы, непоследовательные названия каналов — всё это дробит данные.

3. **Tracking Plan**:
   - Документ-источник истины для всех событий в продукте. Структура: Event Name | Trigger | Parameters | Platform | Owner | Status.
   - Event naming convention: snake_case, формат `[объект]_[действие]`. Примеры: `button_clicked`, `form_submitted`, `checkout_started`, `trial_started`, `feature_used`, `page_viewed`.
   - Параметры события: context (page, section), user properties (plan, cohort), event-specific (item_id, value, currency).
   - Владелец события — конкретный человек (разработчик, маркетолог), не «команда».

4. **Воронки и атрибуция**:
   - Воронка: определить ключевые шаги конверсии → настроить события на каждом шаге → в GA4 Explorations/Funnel Exploration смотреть drop-off → искать причину (технические баги, UX, оффер).
   - Атрибуция моделей: Last Click (по умолчанию в GA4), First Click, Linear, Time Decay, Position Based, Data-Driven (нужно от 700 конверсий/мес). Никогда не сравнивай данные разных платформ напрямую — у каждой своя модель.
   - Независимая атрибуция (Northbeam, Triple Whale, Rockerbox) — если нужно свести данные из рекламных кабинетов и GA4 в одну модель.

5. **Дашборды и отчётность**:
   - Структура дашборда: KPI по каналам (CPL, CR, ROAS) → воронка конверсий → когортный анализ (LTV) → retention (Day 1/7/30).
   - GA4 Looker Studio: коннектор через Data Studio GA4 connector или BigQuery. Для реального времени — GA4 Realtime API.
   - Сигналы проблем: резкое падение событий (тег сломался), аномально высокий CR (дублирование конверсий), расхождение GA4 vs рекламный кабинет >20% (проблема с атрибуцией или фильтрами).

GTM: ИНСТРУКЦИИ ПО НАСТРОЙКЕ КЛЮЧЕВЫХ СОБЫТИЙ

1. `form_submitted` — отправка формы

Data Layer (добавить в код при submit):
window.dataLayer = window.dataLayer || [];
dataLayer.push({
  event: 'form_submitted',
  form_id: 'contact_form',
  form_name: 'Форма обратной связи',
  page_url: window.location.href
});

GTM — Tag:
- Type: Google Analytics: GA4 Event
- Event name: `form_submitted`
- Parameters: `form_id` → `{{DL - form_id}}`, `form_name` → `{{DL - form_name}}`

GTM — Trigger:
- Type: Custom Event
- Event name: `form_submitted`

GTM — Variables (создать для каждого параметра):
- Type: Data Layer Variable
- Data Layer Variable Name: `form_id` / `form_name`
