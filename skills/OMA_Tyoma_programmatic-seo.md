## DOMAIN KNOWLEDGE

1. **Суть программатик SEO:** создавать сотни и тысячи SEO-страниц по формуле «шаблон + данные». Каждая страница закрывает отдельный поисковый запрос. Примеры: Zapier — «[X] integrates with [Y]» (тысячи интеграционных страниц), Tripadvisor — «Рестораны в [город]», Ahrefs — «Backlink checker for [domain]». Ключ — масштаб при минимальных ручных усилиях.

2. **Типы pSEO-страниц и когда что использовать:**
   - Location pages: «[сервис] в [город]» — работает для бизнесов с географическим охватом
   - Comparison pages: «[A] vs [B]» — перехватывают аудиторию на стадии выбора
   - Integration pages: «[tool] + [tool]» — модель Zapier, работает в B2B SaaS
   - Directory pages: «Лучшие [категория] в [нише]» — агрегаторный формат
   - «Alternatives to» pages: «Альтернативы [продукт]» — перехват конкурентного трафика
   - Template/tool pages: «[Шаблон/инструмент] для [задачи]» — работает в productivity/маркетинге

3. **Условие успеха — уникальная ценность каждой страницы:** подстановка слова в шаблон — это thin content (тонкий контент). Google штрафует. Уникальность достигается через уникальные data points на каждой странице: реальные данные по городу/инструменту/категории, отзывы, цены, рейтинги, локальные факты. Правило: если убрать переменную и страница всё равно читается одинаково — страница невалидна.

4. **Источники данных:** собственная база (CRM, каталог товаров, пользовательский контент), публичные API (Google Places, OpenStreetMap, GitHub, Product Hunt, G2), ручной сбор (CSV/Google Sheets), партнёрские данные, веб-сбор данных (где легально). Качество данных = качество страниц. Мусорные данные → мусорные страницы → штраф.

5. **Дизайн шаблона:** H1 с ключевым словом, уникальный intro-параграф (не копипаст — генерируется из данных), structured data (Schema.org под тип страницы), внутренние ссылки между связанными страницами (например, все города в стране ссылаются на страницу страны), явный CTA. Шаблон должен иметь переменные минимум для: заголовка, intro, 2–3 уникальных блоков с данными, мета-тегов.

6. **Техническая архитектура pSEO:**
   - URL: чистая структура — /integrations/slack-notion/, /locations/moscow/, /compare/figma-vs-sketch/
   - Canonical URLs: обязательно, особенно при пагинации
   - XML sitemap: отдельная карта для pSEO-секции, обновляется при добавлении страниц
   - Robots.txt: закрывать страницы с низкой ценностью (фильтры, дубли, тестовые)
   - Crawl budget: при тысячах страниц — приоритизировать через Internal PageRank (больше внутренних ссылок = больше краулинга)
   - Пагинация: rel=next/prev или infinite scroll с SSR — только не клиентский рендеринг

7. **Приоритизация страниц:** формула — Search Volume × (1 / Keyword Difficulty) × Business Value. Сначала строить страницы с наибольшим объёмом запросов и наименьшей конкуренцией. Инструменты для оценки: Ahrefs Keywords Explorer, Semrush, Google Keyword Planner. Минимальный первый батч — 10–20 страниц для тестирования шаблона перед масштабированием.

8. **Риски и как их закрывать:**
   - Duplicate content → решается через уникальные data points и динамический intro-текст, а не спинтакс
   - Crawl budget waste → robots.txt + приоритизация internal links
   - Некачественные данные → валидация перед публикацией, дата обновления на странице
   - Массовые деиндексации → не публиковать страницы без минимум 3 уникальных data points

9. **Метрики pSEO-кампании:** Indexed pages (%) — сколько из созданных страниц в индексе Google, Organic traffic per page (средний), Ranking positions по целевым ключам, CTR из Google Search Console, Conversion rate с pSEO-страниц. Первая проверка — через 4–6 недель после индексации.

### ЭТАП 0: Диагностика размытого запроса

Когда включается:
- Пользователь говорит что-то вроде «мне нужно SEO» или «хочу больше трафика», но без конкретики.
- Задача явно не сформулирована: нет типа сайта, нет понимания что именно строить.
- Запрос содержит противоречие (например, «хочу тысячу страниц, но у меня нет данных»).

Что делать:
- Мягко зафиксировать, что запрос пока общий, и что сейчас разберёмся.
- Задать 2–3 диагностических вопроса:
  - «Какой конкретный результат хочешь получить — трафик, лиды, видимость по определённым запросам?»
  - «Что уже есть: сайт, база данных, список ключевых слов?»
  - «Ты уже знаком с программатик SEO или это первый раз?»
- Сформулировать гипотезу задачи: «По описанию похоже, что тебе нужен [X], а не [Y]. Верно?»
- После подтверждения → Этап 1.

в) «Есть ли уже какие-то данные — каталог, список городов/партнёров/продуктов, или нужно всё строить с нуля? — чтобы понять, с чем работаем, и не предлагать то, чего нет.»
