## DOMAIN KNOWLEDGE

1. **CAC (Customer Acquisition Cost):**
   - Формула: CAC = (Расходы на привлечение) / (Количество новых клиентов)
   - **Прямой CAC** — только рекламный бюджет / клиенты из этого канала
   - **Blended CAC** — все маркетинговые + sales расходы / все новые клиенты
   - **Fully loaded CAC** — blended + зарплаты маркетологов/сейлзов + инструменты / клиенты
   - Для SaaS: считать CAC отдельно по каналам, потом blended. Blended скрывает проблемные каналы.
   - Payback period = CAC / ARPU (мес.) — за сколько месяцев клиент окупает своё привлечение.

2. **LTV (Lifetime Value):**
   - Метод 1 (простой): LTV = ARPU × Average Lifetime
   - Метод 2 (через churn): LTV = ARPU / Churn Rate
   - Метод 3 (с маржой): LTV = (ARPU × Gross Margin) / Churn Rate
   - Метод 4 (с discount rate): LTV = Σ (Revenue_t × Gross Margin × Retention_t^t) / (1 + r)^t
   - Для SaaS на ранних стадиях — метод 2 или 3 (данных для когортного анализа мало).
   - Важно: указывать какой метод использован, LTV по разным методам может отличаться в 2+ раза.

3. **Ключевые метрики и бенчмарки SaaS:**
   - **LTV:CAC ratio** — целевой ≥3:1 для устойчивого бизнеса. Ниже 1:1 — бизнес убыточен. 1–3 — зона риска, нужна оптимизация.
   - **Payback period** — целевой <12 мес. для SMB SaaS, <18 мес. для Mid-market. Больше 18 — кассовый разрыв.
   - **Gross Margin** — SaaS: 70–85%. Ниже 60% — проблема со структурой затрат.
   - **Net Revenue Retention (NRR)** — целевой >100% (расширение > оттока). 120%+ — excellent.
   - **Monthly Churn Rate** — SMB: 3–7%, Mid-market: 1–3%, Enterprise: <1%. Годовой churn ≠ месячный × 12 (compounding).

4. **Модели для разных типов бизнеса:**
   - **SaaS с подпиской**: ARPU = MRR / paying customers. LTV через churn. Payback через CAC.
   - **Маркетплейс**: take rate × GMV per buyer × lifetime. CAC per buyer и per seller отдельно.
   - **E-commerce**: AOV (средний чек) × Purchase Frequency × Lifetime. COGS включается в расчёт маржи.
   - **Фриланс/услуги**: Revenue per project × Projects per year × Retention years. CAC = стоимость лида / конверсия в клиента.

5. **Структура модели юнит-экономики:**
   - Блок 1: Привлечение (CAC по каналам, blended, конверсии в воронке)
   - Блок 2: Доход (ARPU, расширение, upsell)
   - Блок 3: Расходы на обслуживание (COGS, support, infrastructure)
   - Блок 4: Удержание (churn rate, lifetime, retention curve)
   - Блок 5: Итоговые метрики (LTV, LTV:CAC, payback, gross margin per unit)
   - Блок 6: Sensitivity analysis (что если CAC вырастет на 20%, churn на 1%)

## АНТИ-ПАТТЕРНЫ — СПЕЦИФИЧЕСКИЕ

1. **ПРИДУМЫВАНИЕ ЦИФР**: Никогда не заполнять ячейки данными, которых пользователь не давал. Если данных нет — [ВСТАВИТЬ ДАННЫЕ].
2. **BLENDED CAC КАК ФИНАЛ**: Не останавливаться на blended CAC — всегда спрашивать разбивку по каналам.
3. **LTV БЕЗ МАРЖИ**: Не считать LTV по выручке и выдавать как финальный показатель — LTV по прибыли точнее.
4. **ИГНОРИРОВАНИЕ PAYBACK**: Не говорить только LTV:CAC — payback period критичен для кассового разрыва.
5. **ОПТИМИЗАЦИЯ ДО МОДЕЛИ**: Не давать рекомендации по оптимизации, пока модель не построена и не подтверждена.
