## DOMAIN KNOWLEGE

1. **ФРЕЙМВОРКИ КВАЛИФИКАЦИИ:**
   — **BANT**: Budget (есть ли бюджет), Authority (ЛПР ли собеседник), Need (есть ли потребность), Timeline (когда планируют). Простой, для transactional sales.
   — **MEDDIC**: Metrics (какие метрики улучшит), Economic Buyer (кто платит), Decision Criteria (по чему выбирают), Decision Process (как принимают решение), Identify Pain (какая боль), Champion (кто внутри продвигает). Для enterprise sales.
   — **CHAMP**: CHallenges (вызовы), Authority (власть), Money (бюджет), Prioritization (приоритет). Альтернатива BANT с фокусом на боль.
   — Выбор: <$5k ACV → BANT, $5k–$50k → CHAMP, >$50k → MEDDIC.

2. **MQL vs SQL vs SAL:**
   — **MQL (Marketing Qualified Lead)**: Соответствует ICP + проявил интерес (скачал, зарегистрировался, посетил pricing). Определяет маркетинг.
   — **SAL (Sales Accepted Lead)**: SDR подтвердил MQL, начал outreach. Переходная стадия.
   — **SQL (Sales Qualified Lead)**: SDR провёл квалификацию, подтвердил BANT/MEDDIC, готов к демо. Определяет SDR.
   — Конверсия MQL→SQL benchmark: 13–25% (HubSpot, 2024).

3. **СКОРИНГ-МОДЕЛЬ:**
   — **Фирмографические баллы**: Отрасль (+20), размер компании (+15), география (+10), должность (+25).
   — **Поведенческие баллы**: Посетил pricing (+30), скачал гайд (+15), открыл email (+5), посетил 3+ страниц (+20).
   — **Порог MQL**: 50+ баллов. **Порог SQL**: 80+ баллов + firmographic fit.
   — Пересмотр: каждые 3 месяца — калибровать баллы по реальным конверсиям.

4. **СИГНАЛЫ КАЧЕСТВЕННОГО ЛИДА:**
   — Соответствует ICP (фирмография)
   — Проявил high-intent поведение (pricing, demo request, повторный визит)
   — Есть pain point (упомянул проблему в разговоре / форме)
   — Есть бюджет (или доступ к бюджету)
   — Есть timeline («рассматриваем на Q2»)

5. **СИГНАЛЫ ПЛОХОГО ЛИДА:**
   — Не соответствует ICP (wrong industry, wrong size)
   — Только research intent (читает блог, не запрашивает demo)
   — Нет ЛПР (разговаривает с intern / стажёром)
   — «Пришлите информацию» без конкретики

## АНТИ-ПАТТЕРНЫ — СПЕЦИФИЧЕСКИЕ

1. **КВАЛИФИКАЦИЯ «НА ГЛАЗ»**: Без критериев каждый SDR квалифицирует по-разному.
2. **ВСЕ БАЛЛЫ РАВНЫ**: Посетить блог ≠ запросить demo. Поведенческие баллы должны отличаться в 5–10 раз.
3. **НЕТ SLA ПЕРЕДАЧИ**: MQL без контакта за 5 минут теряет 80% потенциала.
