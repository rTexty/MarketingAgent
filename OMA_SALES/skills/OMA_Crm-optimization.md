## DOMAIN KNOWLEDGE

1. **PIPELINE STAGES:**
   — Каждый stage = конкретный milestone в продаже, не абстракция.
   — Типовые stages (B2B SaaS):
     • New Lead → Contacted → Qualified → Discovery → Demo → Proposal → Negotiation → Closed Won / Closed Lost
   — Критерии перехода: что должно произойти, чтобы deal перешёл на следующий stage.
   — Win probability per stage: New (5%) → Qualified (15%) → Demo (30%) → Proposal (60%) → Negotiation (80%).

2. **АВТОМАТИЗАЦИИ:**
   — Task creation: при переходе stage → автоматическая задача (позвонить, отправить КП, follow-up).
   — Email sequences: автоматические follow-up при определённых условиях.
   — Lead scoring: автоматический scoring на основе активности.
   — Notifications: уведомления при открытии письма, посещении pricing, неактивности deal.
   — Rotting alerts: deal не двигается N дней → алерт менеджеру.

3. **ОТЧЁТЫ:**
   — Pipeline report: количество и сумма deals по stages.
   — Conversion report: конверсия между stages.
   — Forecast: прогноз закрытия на месяц/квартал.
   — Activity report: количество звонков, писем, встреч по репам.
   — Win/Loss analysis: причины закрытия won vs lost.

4. **ИНТЕГРАЦИИ:**
   — Email: Gmail/Outlook → CRM (автоматическое логирование писем).
   — Calendar: встречи → CRM (автоматическое создание activities).
   — Marketing: формы сайта → CRM (автоматическое создание leads).
   — Support: тикеты → CRM (видеть проблемы клиентов в контексте deals).

## АНТИ-ПАТТЕРНЫ

1. **СЛИШКОМ МНОГО STAGES**: 15+ stages — продавцы не обновляют. 7–9 оптимально.
2. **НЕТ КРИТЕРИЕВ ПЕРЕХОДА**: Stages без критериев — каждый ставит что хочет.
3. **НЕТ АВТОМАТИЗАЦИИ**: Ручные процессы = продавцы тратят время на CRM, а не на продажи.
4. **ОТЧЁТЫ РАДИ ОТЧЁТОВ**: 20 дашбордов, которые никто не смотрит. 3–5 ключевых.
